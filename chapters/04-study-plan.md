# Chapter 4: Study Plan

## 6-Week Study Plan

### Week 1: Fundamentals
- Day 1: Read GitHub Actions overview and architecture.
- Day 2: Create your first workflow for build/test on `push`.
- Day 3: Learn workflow file structure and runner types.
- Day 4: Explore branch and path filters.
- Day 5: Practice contexts, expressions, and `if`.
- Day 6: Study common marketplace actions.
- Day 7: Review and document what you learned.

### Week 2: Workflow Syntax and CI Basics
- Day 1: Deep dive YAML syntax and event triggers.
- Day 2: Build a multi-job CI workflow.
- Day 3: Add matrix builds for multiple runtime versions.
- Day 4: Upload artifacts and test outputs.
- Day 5: Use `needs` for job ordering.
- Day 6: Practice workflow debugging.
- Day 7: Refine your workflows.

### Week 3: Reusable Workflows and Variables
- Day 1: Learn `env`, `secrets`, and workflow inputs.
- Day 2: Implement a reusable workflow.
- Day 3: Build a composite action for shared logic.
- Day 4: Call reusable workflows from another repository.
- Day 5: Practice outputs and cross-job data passing.
- Day 6: Study action metadata files.
- Day 7: Review reusable workflow patterns.

### Week 4: Runners, Security, and Caching
- Day 1: Compare GitHub-hosted and self-hosted runners.
- Day 2: Set up a self-hosted runner on a VM or cloud host.
- Day 3: Learn workflow security best practices.
- Day 4: Configure environments, protections, and approvals.
- Day 5: Implement artifact upload/download and caching.
- Day 6: Practice cache key design and restore operations.
- Day 7: Document security and runner learnings.

### Week 5: Containers, Deployment, and Releases
- Day 1: Build a container-based workflow.
- Day 2: Create and publish a Docker image.
- Day 3: Automate deployment to a cloud platform.
- Day 4: Deploy to Kubernetes with a manifest.
- Day 5: Automate releases and package publishing.
- Day 6: Practice GitHub Packages and release uploads.
- Day 7: Review deployment workflows.

### Week 6: Monitoring and Exam Prep
- Day 1: Study workflow logs, annotations, and summaries.
- Day 2: Troubleshoot failed jobs and reruns.
- Day 3: Optimize workflow execution time.
- Day 4: Take practice exams or quizzes.
- Day 5: Review and strengthen weak topics.
- Day 6: Create a final exam preparation summary.
- Day 7: Do a final review of key concepts.

## Alternative 4-Week Plan
Compress the same topics into four weeks by combining related exercises and increasing practice frequency. Focus on fundamentals for the first two weeks and advanced automation plus exam review in the last two.
